# frontend-plan.md — مهام تنظيف HTML و CSS و JavaScript

**المجلد:** `Frontend/`  
**المرجع:** نتائج التدقيق في `docs/architecture.md` (2026-04-12).

---

## أهداف عامة (ملخص)

- توحيد لون الهيدر إلى `#0a5c56` على كل الصفحات والـ manifest حيث ينطبق.
- إزالة اعتماد Tailwind عبر CDN وتوحيد النموذج على CSS محلي تحت `assets/css/`.
- تقليل التكرار: مصدر واحد للأنماط المشتركة؛ إزالة أو دمج روابط CDN غير مستخدمة.
- فصل JavaScript والأنماط المضمّنة إلى ملفات خارجية مع ترتيب تحميل واضح.

---

## ترتيب الأولويات (من الأعلى إلى الأدنى)

### 1) الأولوية القصوى — تنظيف الكود وتوحيده

#### 1.1 توحيد لون الهيدر `#0a5c56` وإزالة Tailwind / CDN غير المستعمل

- [ ] جرد كل الصفحات التي تربط `theme.css` دون `iag-theme.css` أو التي تعرّف ألوان هيدر داخل `<style>` (`#0f172a`, `#020617`, `#0f766e`, …) ومطابقتها مع القاعدة.
- [ ] حل التعارض بين `theme.css` (`.page-header` بـ `#0f172a`) و`iag-theme.css` (`#0a5c56`): ترتيب الربط أو دمج القواعد بحيث لا يطغى لون على الآخر دون قصد.
- [ ] تحديث `manifest.json` (`theme_color` / `background_color`) من `#0f766e` إلى `#0a5c56` أو قيم متسقة مع الهوية المعتمدة.
- [ ] إزالة `<script src="https://cdn.tailwindcss.com">` من الصفحات التي لا تعتمد فعلياً على utilities Tailwind؛ استبدال أي فئات مستخدمة بقواعد في ملفات CSS محلية.
- [ ] مراجعة `https://unpkg.com/lucide@latest` (أو المسارات المماثلة): تثبيت نسخة معروفة محلياً أو عبر مسار ثابت لتقليل مخاطر سلسلة التوريد.
- [ ] الإبقاء على الاعتماديات المبررة فقط (مثل خط Cairo من Google Fonts، أو DataTables في `admin.html` إن بقيت مطلوبة) وتوثيق سبب كل CDN متبقٍ.

#### 1.2 فصل CSS المضمّن (`<style>`) إلى ملفات خارجية

- [ ] إنشاء أو توسيع ملفات تحت `assets/css/` (مثلاً `layout.css`, `components.css`) وفق تجميع منطقي (هيدر، بطاقات، قوائم جانبية، أزرار).
- [ ] لكل صفحة رئيسية (`index`, `settings`, `notifications`, `forms`, `findings`, `dashboard`, `employee`, `coordinator`, `distribution`, `admin`, `car-dashboard`, `car`, `portal`, `portal_car`, `phc_form`): نسخ القواعد من `<style>` إلى الملف الخارجي ثم حذف البلوك بعد التحقق بصرياً.
- [ ] توحيد مسار `card.css`: إما نقله إلى `assets/css/` وتحديث المراجع، أو توحيد الاستيراد من جذر `Frontend/` مع توثيق القرار (حالياً `employee.html` فقط).
- [ ] مراجعة `styles.css` غير المستورد حالياً: دمجه أو حذفه أو ربطه من الصفحات لتجنب «ملف ميت».

#### 1.3 فصل JavaScript المضمّن (`<script>` بدون `src`) إلى `assets/js/`

- [ ] لكل صفحة تحتوي منطقاً مضمّناً: إنشاء `assets/js/<اسم-الصفحة>.js` (أو وحدة مشتركة + ملف صفحة) ونقل الدوال والتهيئة مع الحفاظ على ترتيب التبعيات (`config.js`, `auth.js`, `nav.js`).
- [ ] استبدال البلوك الداخلي بـ `<script src="assets/js/... defer">` أو ترتيب صريح بعد المكتبات المطلوبة.
- [ ] حالات خاصة من التقرير:
  - [ ] `car.html`: توحيد مسار التوجيه الفوري (`location.replace`) مع بقية التطبيق وتجنب تحميل محتوى زائد بعد التحويل.
  - [ ] `portal_car.html`: توحيد مصدر عنوان الـ API مع `CONFIG.API_URL` (بدلاً من `localStorage` الفارغ فقط) مع سلوك واضح عند غياب الإعداد.

### 2) أولوية عالية — اتساق الواجهة والصيانة

- [ ] ضمان أن كل الصفحات الداخلية تربط نفس مجموعة الأصول الأساسية (`iag-theme.css` + ملفات التخطيط المشتركة) حيث ينطبق.
- [ ] مراجعة `Frontend/restyle.js`, `Frontend/sw.js`, `Frontend/card.js`: إما ربطها من الصفحات/manifest أو إزالتها/أرشفتها بعد التأكد من عدم الحاجة إليها.
- [ ] توثيق ترتيب تحميل CSS/JS في تعليق قصير في `docs/` أو تعليق رأس مشترك عند الاتفاق على المعيار.

---

### 3) تحسين الأداء

- [ ] تقليل حجم وعدد طلبات الشبكة: دمج ملفات CSS/JS الصغيرة حيث لا يضر بالتخزين المؤقت؛ تأجيل تحميل غير الحرج.
- [ ] مراجعة الصور والأصول الثابتة (إن وُجدت) والضغط/التخزين المؤقت المناسب لبيئة الاستضافة (GitHub Pages أو غيرها).
- [ ] بعد إزالة Tailwind CDN: قياس زمن التحميل الأولي مقارنةً بالوضع الحالي (اختياري لكن مفيد للتحقق).

---

## ترتيب تنفيذي مقترح (خطوات عملية)

1. جرد `<head>` لكل HTML: روابط CSS/JS وCDN والـ inline.
2. توحيد الهيدر والألوان في ملفات CSS خارجية ثم إزالة التعارض مع `theme.css` / `iag-theme.css`.
3. إزالة Tailwind CDN واستبدال الاستخدامات الفعلية بـ CSS محلي.
4. نقل `<style>` صفحة بصفحة مع اختبار بصري لكل صفحة.
5. نقل `<script>` الداخلي إلى `assets/js/` مع اختبار تدفق `fetch`/`post` لكل دورة مستخدمة.
6. تنظيف الملفات غير المستخدمة وتحسين الأداء كخطوة أخيرة لهذه المرحلة.
